# Imbas OS demo script

This is a draft demo flow for private previews, videos, or early-access walkthroughs.

## Goal

Show that Imbas OS turns scattered AI work into durable, reviewable, reusable operating context.

## 3-minute demo

### 1. Start in Command Center

Say:

> This is Imbas OS: a local-first home base for AI agents. It tracks artifacts, wiki knowledge, agent runs, proposals, mobile sessions, and memory migration state.

Show:

- module health;
- artifact/wiki/run/proposal counts;
- recent Runledger/Lorekeeper panels.

### 2. Use Agent Console

Say:

> Instead of leaving agent work in chat history, I can route it through the Agent Console and record what happened.

Do:

- send a bounded OpenClaw task;
- show reply action card;
- replay Runledger or save reply as artifact.

### 3. Save or open an artifact

Say:

> Generated artifacts become local bundles with metadata, notes, snapshots, and safe replay.

Show:

- artifact view;
- sandbox/trust boundary;
- notes/provenance.

### 4. Create a Lorekeeper proposal

Say:

> Durable knowledge changes go through review. Agents propose; humans approve.

Show:

- proposal rationale;
- markdown preview;
- before/after diff;
- approve/apply guard.

### 5. Show snapshots and restore

Say:

> Before managed wiki changes apply, Imbas snapshots the previous markdown. Restore is guarded and audited.

Show:

- snapshot list;
- preview snapshot;
- explain explicit RESTORE confirmation without doing a risky live restore unless using demo data.

### 6. Show Android companion

Say:

> The phone app is a pocket companion: pair, check diagnostics, review Runledger, approve/reject proposals, and capture notes.

Show:

- pairing QR/challenge;
- Diagnostics tab;
- Runs tab filtering;
- Capture tab.

### 7. Close with the value

Say:

> The point is simple: AI work should not evaporate when the chat ends. Imbas OS gives it a local, reviewable, reusable home.

## Do not claim yet

- public release readiness;
- production security certification;
- enterprise compliance;
- hosted cloud availability;
- team collaboration availability;
- MemPalace retirement.
