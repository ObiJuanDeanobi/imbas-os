## Problem section

### AI work disappears too easily

AI agents can draft plans, build tools, review code, summarize evidence, and make decisions — but their work often gets stranded in chat history, terminal scrollback, download folders, and scattered Markdown files.

That creates three problems:

- useful outputs vanish;
- context has to be rebuilt every session;
- human review and audit trails are bolted on later, if they exist at all.

## Product section

### One local-first workspace for agent work

Imbas OS brings the pieces together:

- generated artifacts and mini-tools;
- living Markdown/wiki pages;
- agent memory and context events;
- context packs for future runs;
- Runledger audit history;
- Lorekeeper review proposals;
- snapshots and restore;
- Android companion capture and review.

## Trust section

### Local-first by default

The core is designed to run locally. Your artifacts, wiki, memory, run history, and approvals belong to you.

Future hosted/cloud features should add convenience — sync, backups, collaboration, managed connectors — without breaking the local-first core.

## Review section

### Human review is part of the workflow

Agents should not silently rewrite your source of truth.

Lorekeeper uses proposal-first wiki updates, visual diffs, approvals, snapshots, restore, and Runledger evidence so important changes stay inspectable.

## Agents section

### Bring your agents

Imbas OS starts with OpenClaw and is designed around connector boundaries for Hermes, Codex, Claude Code, and future agents.

The goal is not one more chat window. The goal is a shared operating context agents can write to, search, and use safely.

## Early access section

### Early access for builders who want the sharp edge

The local core will be free/open source. Patreon supporters can help fund development and get early preview builds, dev notes, and a closer feedback loop.

Paid cloud/workspace features will be separate later: hosted sync, backups, collaboration, admin controls, compliance exports, and managed connectors.

## Demo section

### The 3-minute demo

1. Ask an agent to research or build something.

3. Propose a Lorekeeper wiki update.
4. Preview the diff and approve/apply.
